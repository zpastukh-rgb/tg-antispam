# Этапы разработки AntiSpam Guardian

По ТЗ 1 и ТЗ 2. Порядок — от MVP к расширенному продукту.

---

## Этап 1 — MVP: панель, пользователь, лимиты (текущий фокус)

**Цель:** Стабильная панель с разделением сущностей, пользователь с тарифом и лимитом чатов, без реальной оплаты.

### 1.1 Модели и БД ✅
- [x] Модель **User**: telegram_id, username, first_name, subscription_until, tariff (FREE/PRO/BUSINESS), chat_limit, created_at, is_admin, status.
- [x] Модель **Payment** (задел под оплату): user_id, amount, currency, months, tariff, created_at, expires_at, status, provider, payment_id.
- [x] В **Chat** поле **is_log_chat** (bool): группа зарегистрирована как лог-чат через /setlog.
- [x] Разделение: защищаемые чаты (is_log_chat=False), лог-чаты (is_log_chat=True).

### 1.2 Пользователь и лимиты
- [x] get_or_create_user(telegram_id, …): создание/обновление User, тариф FREE, chat_limit=1 по умолчанию.
- [x] При **/check**: проверка «подключённых чатов < chat_limit»; при превышении — сообщение «Нужен тариф выше».
- [x] На главном экране панели: тариф, «чатов X из Y», дата окончания подписки.

### 1.3 Панель (по ТЗ 2)
- [x] **Главный экран** — 5 кнопок: Защита | Отчёты | Чаты | Тариф и оплата | Подключить чат.
- [x] **Вложенное меню «Защита»**: Фильтры | Наказания | Новички | Стоп-слова | Анти-рейд (заглушка «на PRO»).
- [x] Фильтры / Наказания / Новички / Стоп-слова — текущая логика, перенесённая под «Защита».
- [x] Схема callback: panel:main, panel:protection, panel:protection:filters и т.д. (по ТЗ 2).

### 1.4 Чаты и отчёты
- [x] **Чаты** → «Подключённые чаты»: только защищаемые (is_log_chat=False).
- [x] **Чаты** → «Лог-чаты»: только чаты с is_log_chat=True.
- [x] **Отчёты** → «Куда слать»: только лог-чаты пользователя (где был выполнен /setlog).
- [x] Инструкция «Как работает»: создать группу → добавить бота → права → /setlog → в панели выбрать в «Куда слать».

### 1.5 Команда /setlog
- [x] В группе команда **/setlog**: только для админа; регистрирует текущий чат как лог-чат (Chat с is_log_chat=True, owner_user_id=вызвавшего).
- [x] Ответ: «Эта группа теперь лог-чат. В панели выбери её в Отчёты → Куда слать.»

### 1.6 Мелкие правки
- [ ] Убрать кнопку пагинации «1/1», если страница одна.
- [ ] На главном экране всегда показывать: текущий чат, лог-чат для него, тариф, лимит.

## Этап 2 — Оплата и анти-рейд

**Цель:** Реальная монетизация и защита от рейдов.

### 2.1 Оплата
- Интеграция с провайдером (ЮKassa / Telegram Payments / CryptoBot и т.д.).
- Выбор периода: 1 / 3 / 6 / 12 / 24 месяца.
- После оплаты: обновление subscription_until, смена tariff и chat_limit.
- Экран «История оплат» в панели.

### 2.2 Истечение подписки
- Периодическая проверка subscription_until; при истечении — перевод на FREE, chat_limit=1, уведомление.
- Ограничение действий при просрочке (например, только просмотр панели или напоминание оплатить).

### 2.3 Анти-рейд (PRO/BUSINESS)
- Настройки в панели: ВКЛ/ВЫКЛ, порог (N новых участников), окно (секунды), действие (закрыть вход / мут и т.д.).
- Логика: при my_chat_member считаем «новых за окно»; при превышении порога — выполнить действие.
- Учёт только для чатов с включённым анти-рейдом и при тарифе PRO/BUSINESS.

### 2.4 Статистика (по желанию)
- «За неделю удалено N сообщений», «Предотвращено N рейдов» — в панели и/или в отчётах.

## Этап 3 — Рост и виральность

**Цель:** Увеличение охвата и удержания.

### 3.1 Публичные сообщения
- Режимы: тихий | умеренный | публичный.
- В публичном режиме бот может писать в чат короткие фразы («Guardian уничтожил спам», «Режим шторма» и т.д.).

### 3.2 Кнопки после /check
- «Добавить в другой чат», «Открыть панель» (уже есть частично).
- «Поделиться ботом» — шаблон сообщения / ссылка.

### 3.3 Напоминания и воронка
- Напоминание, если пользователь начал подключение и не завершил (через 12/24 ч).
- Опционально: квалификационные вопросы (сколько чатов, размер) и предложение тарифа.

### 3.4 Guardian Levels (позже)
- Уровни бота по «предотвращённому спаму»; косметика и геймификация.

---

## Порядок работ (кратко)

| Этап | Содержание |
|------|------------|
| **1** | User, Payment, Chat.is_log_chat → панель 5 кнопок + Защита → разделение чатов/лог-чатов → /setlog → лимит при /check |
| **2** | Платёжный провайдер → продление подписки → истечение подписки → анти-рейд → статистика |
| **3** | Публичные сообщения → share/invite → напоминания → уровни Guardian |

---

## Технические принципы (не ломать)

- **Модерация:** pipeline evaluate() → apply_action() → send_log().
- **Память:** bounded cache (LRU + TTL) для панели, стоп-слов, whitelist, newbie.
- **Роутеры:** команды (/check, /start, /health, /setlog) обрабатываются до moderation_router.
- **Структура:** разнесение по файлам (handlers, db, utils), без монолитного слияния.
